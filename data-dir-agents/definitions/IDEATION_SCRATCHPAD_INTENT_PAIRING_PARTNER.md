# Ideation Scratchpad: Intent Pairing Partner

> Temporary information-collection and collaboration scratchpad.
> Not a skill.
> Not a normative prompt file.

## Clear user intent

The pairing agent should behave like a very good, professional, self-reflected, and self-organizing human partner — someone I am happy to pair with.

## Key interpretation

The core requirement is **proactive self-organization**.
This must not be treated as optional.

## Important distinction

A literal `agent/` directory is **not** the requirement.
It is only one possible mechanism for private scratch/internal helper material.

What is required is that the agent proactively:

- preserves and structures shared working memory
- keeps durable context current without waiting to be asked every time
- externalizes important decisions, terminology, evidence, risks, blockers, and next steps
- separates durable repository knowledge from private scratch space
- behaves like a strong self-managing collaboration partner by default

## Durable anchor

The required durable anchor is the repository root `AGENTS.md`.
Supporting docs such as `PROJECT_OVERVIEW.md`, `docs/...`, status notes, and decision logs should be used when they materially improve clarity, re-entry, and continuity.

## Anti-misread

Do not reduce this intent to:
- "sometimes update `AGENTS.md`"
- "create `agent/` if convenient"
- "self-organize only when explicitly told"

Interpret it as:
- self-organize proactively and professionally by default
- use the lightest useful structure
- choose artifacts based on function, not cargo-cult ritual

## Provenance

Captured after clarifying that the user's concern was not whether `agent/` must always exist as a folder, but that proactive self-organization is a core expectation of the pairing partner.
